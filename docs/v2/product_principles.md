# Hero Performance OS Product Principles

## Purpose

`Hero Performance OS` is not a broad consumer poker app.
It is a personal performance system designed to make Hero better as:

- a semipro poker player,
- a data engineer,
- and an AI product engineer.

The product should become more useful to Hero before it becomes more marketable to others.

## Principles

1. Results are downstream; decision quality comes first.
2. Session packets are cumulative evidence, not isolated review prompts.
3. Baseline comparison matters more than generic one-shot judgment.
4. Track drift, not just leaks.
5. Separate field distortion from hero contamination.
6. Preserve positive execution as first-class memory.
7. Use deterministic evidence and confidence before narrative compression.
8. Use LLMs for explanation support, not canonical truth decisions.
9. Keep the operator loop as part of the product, not a temporary workaround.
10. The system must sharpen Hero as a player, builder, and analyst at the same time.

## MVP questions

The V2 MVP should answer six questions reliably:

1. Which hand classes are underperforming relative to Hero's baseline?
2. Where is Hero's style drifting?
3. Which previously stable strengths are regressing?
4. What field distortions characterize the current sample?
5. Where is Hero adapting well versus becoming contaminated by the field?
6. What is the next best adjustment before the next session?
